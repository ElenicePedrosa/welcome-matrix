Algoritmo como processo está inserido em qualquer contexto.

Quando se aprende algoritmos de verdade, a linguagem de programação escolhida para o desenvolvimento vira um mero detalhe.

Eu não conhecia o Diagrama de Chapin como uma representação gráfica de algoritmos. Acho o fluxograma e o pseudocódigo mais fáceis de se entender.

O fluxograma apresentado nos exercícios é bem parecido com o desenvolvimento dentro de Outsystems. A condicional IF inclusive utiliza o mesmo losango como representação visual.

É importante que se reflita sobre os fluxos e variáveis que serão utilizadas para resolver algum problema antes de iniciar a codificação.
